üîê Core Principles & Fundamentals
CIA Triad - Foundation of Information Security
Confidentiality
Ensuring information is accessible only to authorized individuals, entities, or processes
Real-world Example: Medical records encrypted in a hospital database where only authorized doctors and nurses can access patient information using role-based access controls.
Data Classification
Categorizing data based on sensitivity (Public, Internal, Confidential, Restricted)
Access Controls
Mechanisms to restrict access (DAC, MAC, RBAC, ABAC)
Integrity
Maintaining accuracy and completeness of data throughout its lifecycle
Real-world Example: Digital signatures on software updates ensuring the code hasn't been tampered with during distribution from the vendor to end users.
Data Validation
Input validation, checksums, hash functions
Version Control
Change tracking and audit trails
Availability
Ensuring information and resources are accessible when needed by authorized users
Real-world Example: Netflix's distributed architecture with multiple data centers ensuring 99.9% uptime even during peak traffic or system failures.
Redundancy
Backup systems, failover mechanisms, load balancing
Disaster Recovery
Business continuity planning, recovery procedures
Anonymity
Protecting the identity of individuals in communications and transactions
Real-world Example: Tor browser allowing users to browse the internet without revealing their IP address or location to websites or network observers.
Traffic Analysis Resistance
Preventing correlation attacks through timing and volume analysis
Identity Unlinkability
Ensuring actions cannot be linked to specific individuals
üîë Identity & Access Management (IAM)
Authentication & Identity Components
Identity
A unique representation of a user, device, or service within a system
Real-world Example: Your Google account identity includes your email, profile information, and associated services, allowing Google to recognize you across Gmail, Drive, and YouTube.
Digital Identity
Online representation including usernames, email addresses, certificates
Identity Federation
Single Sign-On (SSO) across multiple systems
Credentials
Evidence used to verify identity during authentication processes
Real-world Example: Banking app using multiple credentials: your username/password (something you know), SMS code (something you have), and fingerprint (something you are).
Multi-Factor Authentication (MFA)
Something you know, have, and are
Credential Management
Password policies, credential rotation, secure storage
Claims
Assertions about a user's identity, attributes, or permissions
Real-world Example: JWT token containing claims like user role ("admin"), department ("engineering"), and expiration time, allowing microservices to make authorization decisions.
Security Tokens
JWT, SAML tokens containing encoded claims
Attribute-Based Access Control
Authorization based on user attributes and environmental factors
Public Key Infrastructure (PKI)
Framework for creating, managing, and validating digital certificates and public-private key pairs
Real-world Example: HTTPS certificates on websites verified by Certificate Authorities like Let's Encrypt, ensuring secure communication between browsers and web servers.
Public Key
Openly shared cryptographic key used for encryption and signature verification
Private Key
Secretly held key used for decryption and digital signing
Salt
Random data added to passwords before hashing to prevent rainbow table attacks
üîí Cryptography & Data Protection
Cryptographic Functions & Protocols
Pseudorandom Functions (PRF)
Efficiently computable functions that appear random to any computationally bounded adversary
Real-world Example: AES encryption using PRFs to generate seemingly random ciphertext from plaintext, making it impossible for attackers to detect patterns.
Key Derivation Functions
PBKDF2, scrypt, Argon2 for generating encryption keys
Stream Ciphers
Using PRF output as keystream for encryption
Hash-based Message Authentication Code (HMAC)
Cryptographic authentication code involving a hash function and a secret key
Real-world Example: API authentication where each request includes an HMAC signature, allowing the server to verify the request hasn't been tampered with and comes from an authorized client.
Message Integrity
Detecting unauthorized alterations to data
Authentication Protocols
Challenge-response mechanisms using HMAC
Oblivious Pseudorandom Functions (OPRF)
Cryptographic protocol allowing computation of PRF without revealing inputs to either party
Real-world Example: Password managers using OPRF to check if your password appears in breach databases without revealing your actual password to the service.
Privacy-Preserving Protocols
Secure multi-party computation applications
Private Set Intersection
Finding common elements without revealing sets
Digital Signatures & Authentication
Standard Digital Signatures
Cryptographic mechanism linking a signature to a specific signer and document
Real-world Example: Software developers signing code releases with RSA or ECDSA signatures, allowing users to verify the software came from the legitimate publisher.
RSA Signatures
Based on integer factorization problem
ECDSA
Elliptic curve-based signatures for better efficiency
Group Signatures
Allow group member to sign on behalf of group while maintaining anonymity within the group
Real-world Example: Anonymous employee feedback systems where any employee can submit signed feedback that's verifiably from a company employee, but doesn't reveal which specific employee.
Group Manager
Entity responsible for group key management
Revocation Mechanisms
Removing compromised members from groups
Ring Signatures
Signature scheme where any member of a group can produce signature on behalf of group without revealing identity
Real-world Example: Monero cryptocurrency transactions using ring signatures to hide the actual sender among a group of possible senders, providing transaction privacy.
Spontaneous Groups
No central authority or pre-setup required
Linkable Ring Signatures
Preventing double-spending while maintaining privacy
Blind Signatures
Signatures where signer cannot see content being signed, providing privacy for signature requestor
Real-world Example: Digital cash systems where a bank signs withdrawal tokens without seeing their serial numbers, enabling untraceable electronic payments.
Unlinkability
Preventing correlation between signature creation and usage
E-voting Applications
Ensuring vote privacy while maintaining verification
Homomorphic Encryption
Fully Homomorphic Encryption (FHE)
Encryption allowing arbitrary computations on encrypted data without decryption
Real-world Example: Cloud computing where a hospital can outsource encrypted patient data analysis to a cloud provider who can run diagnostic algorithms without ever seeing the actual patient information.
Bootstrapping
Technique to refresh ciphertext and enable unlimited operations
Lattice-based Schemes
BGV, BFV, CKKS schemes for different operation types
Somewhat Homomorphic Encryption (SHE)
Supports limited number of operations before noise overwhelms the computation
Real-world Example: Financial institutions performing limited statistical analysis on encrypted customer data for fraud detection without exposing individual transaction details.
Noise Management
Controlling accumulated noise during computations
Depth Limitations
Maximum circuit depth before decryption failure
Partially Homomorphic Encryption (PHE)
Supports only one type of operation (either addition or multiplication) unlimited times
Real-world Example: ElGamal encryption for secure voting systems where encrypted votes can be added together to get final tallies without revealing individual votes.
Additive Schemes
Paillier, ElGamal for addition operations
Multiplicative Schemes
RSA for multiplication operations
Leveled Fully Homomorphic Encryption (LFHE)
FHE with predetermined maximum circuit depth, avoiding bootstrapping complexity
Real-world Example: Genome analysis where researchers can perform specific, predetermined genetic analysis algorithms on encrypted DNA data with known computational requirements.
Circuit Optimization
Minimizing depth for specific applications
Parameter Selection
Choosing security parameters for target depth
Zero-Knowledge Proofs
zk-SNARKs (Zero-Knowledge Succinct Non-Interactive Arguments of Knowledge)
Compact proofs allowing verification of computation correctness without revealing inputs
Real-world Example: Zcash cryptocurrency where users can prove they have sufficient funds for a transaction without revealing their actual balance or transaction history.
Trusted Setup
Initial ceremony to generate proving and verifying keys
Circuit Compilation
Converting computations to arithmetic circuits
zk-STARKs (Zero-Knowledge Scalable Transparent Arguments of Knowledge)
Zero-knowledge proofs without trusted setup, using collision-resistant hash functions
Real-world Example: StarkNet blockchain using zk-STARKs to bundle thousands of transactions into a single proof, dramatically reducing on-chain verification costs while maintaining privacy.
Transparency
No trusted setup required, publicly verifiable randomness
Post-Quantum Security
Resistance to quantum computer attacks
Bulletproofs
Short non-interactive zero-knowledge proofs without trusted setup for arithmetic circuits
Real-world Example: Monero using Bulletproofs to prove transaction amounts are positive without revealing actual amounts, reducing transaction size by 80% compared to previous methods.
Range Proofs
Proving values lie within specific ranges
Aggregation
Combining multiple proofs into single compact proof

Privacy-Enhancing Technologies
Mixers & Tumblers
Services that obscure transaction trails by mixing multiple users' funds together
Real-world Example: Tornado Cash (before regulatory issues) allowed Ethereum users to deposit ETH and later withdraw equivalent amounts to different addresses, breaking the transaction link.
Centralized Mixers
Third-party services handling the mixing process
Decentralized Protocols
Smart contract-based mixing without trusted intermediaries
CoinJoin & CoinSwap/Shuffle
Collaborative transaction protocols where multiple users combine their transactions to obscure ownership
Real-world Example: Wasabi Wallet implementing CoinJoin where multiple Bitcoin users create a single large transaction with multiple inputs and outputs, making it difficult to trace which input corresponds to which output.
Equal Output Values
Using standard denomination amounts to increase anonymity set
Coordination Protocols
Secure multi-party protocols for transaction construction
RingCT (Ring Confidential Transactions)
Combination of ring signatures and confidential transactions to hide both sender and transaction amounts
Real-world Example: Monero transactions using RingCT where observers cannot determine who sent money, who received it, or how much was transferred, providing comprehensive transaction privacy.
Stealth Addresses
One-time addresses for each transaction to hide recipients
Pedersen Commitments
Cryptographic commitments to hide transaction amounts
zk-Rollups
Layer 2 scaling solution using zero-knowledge proofs to batch transactions while maintaining privacy
Real-world Example: Polygon zkEVM processing thousands of Ethereum transactions off-chain, then submitting a single proof to mainnet that validates all transactions without revealing individual transaction details.
State Transitions
Proving correctness of batched state changes
Data Availability
Ensuring transaction data remains accessible for verification
BOLT Protocol (Blind Off-chain Lightweight Transactions)
Privacy-preserving payment channel protocol using blind signatures for anonymous micropayments
Real-world Example: Theoretical implementation for private Lightning Network payments where intermediate routing nodes cannot determine payment amounts or link sender to receiver.
Anonymous Channels
Payment channels with hidden balances and participants
Unlinkable Payments
Preventing correlation of multiple payments through same channel

Detecting anomalies in network traffic patterns
